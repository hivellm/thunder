# Proposal-stage vectors — push/streaming v-next

**These vectors are NOT part of the normative corpus.** They live in this
subdirectory deliberately: all five language corpus loaders glob
`conformance/vectors/*.yaml` **non-recursively** (Go `filepath.Glob`, Rust
`read_dir` one level, TS `readdirSync`, Python `Path.glob("*.yaml")`, C#
`EnumerateFiles` without `AllDirectories`), so nothing here is loaded, asserted, or
counted against the anti-shrink floor. The normative corpus stays at its 39 vectors.

## What they are

Design deliverables for `phase5_push-streaming-design` (task 1.6) — the proposed push
envelope from [`docs/analysis/push-streaming/`](../../../docs/analysis/push-streaming/README.md).
Each is a server-initiated frame `Response{id: PUSH_ID, result: Ok(Map{stream, kind,
data})}` for one reserved `kind`:

| File | `kind` | Demonstrates |
|---|---|---|
| `push-envelope-data.yaml` | `data` | A correlated watch event (`stream` = opening request id, WIRE-051) |
| `push-envelope-progress.yaml` | `progress` | A non-terminal tick with a nested-Map payload |
| `push-envelope-end.yaml` | `end` | Terminal close of a stream (Null payload) |
| `push-envelope-error.yaml` | `error` | Terminal-abnormal close carrying a product error |
| `push-envelope-invalidate.yaml` | `invalidate` | Broadcast (`stream = 0`, WIRE-052) cache invalidation |

## Why they matter before ratification

They prove the central claim of the proposal: **the envelope needs no wire-version
bump**. Every frame here is an ordinary MessagePack `Map` inside the existing
`Ok(Value)` payload, so it round-trips through the frozen wire codec unchanged. This
was verified during authoring: each `frame_hex` was generated by the real `thunder-go`
`wire.EncodeFrame(ResponseOK(PushID, …))`, and each decodes-and-re-encodes to the same
bytes. They are in the same schema (TST-001) as the normative vectors.

## Promotion

When the envelope is coordinated with Synap and ratified (see
[`docs/analysis/push-streaming/05-coordination-and-open-questions.md`](../../../docs/analysis/push-streaming/05-coordination-and-open-questions.md)),
move these files up into `conformance/vectors/` and raise each loader's floor. No byte
changes are expected on promotion — they already pass the wire layer.
