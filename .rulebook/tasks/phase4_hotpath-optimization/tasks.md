## 1. Implementation
- [x] 1.1 **Quantify the fairness confound first** (BEN-001): done via the `thunder-stripped` diagnostic lane (same wire, same client, bare listener — excluded from `Lane::ALL` because it head-of-line blocks, so "beating" it would be meaningless). It refuted the stated hypothesis: at pipelined-1k the server's features cost **nothing** (103,022 vs 102,508) — the bottleneck was the client's write mutex, fixed in `1bdbcb6` (6.5×). With the client fixed the same lane now shows what the features really cost: 34.5% at pipelined-1k, 19.3%/14.0% at the `medium-4KiB` cells we do not win, 3.1% at point-echo d1/c1. **Which feature dominates is still open** — the first ablation was invalid (it modelled `spawn().await`, serializing the read loop, where the server spawns fire-and-forget; it claimed 32.7% where whole-server cost is 3.1%) and was reverted rather than reported. Needs the third variant in 1.2b
- [ ] 1.1b **Instrument fixed (`7a7187d`) — the prerequisite nothing else could proceed without.** The phase review found `phase4_shootout` 1.6 ("Pinned runs", BEN-011) checked off while nothing pinned anything, and `artifact.rs` shipping `governor: "unknown (platform probe lands at T4.3)"` — T4.3 being the very task that closed without it. Landed: CPU pinning of the runtime workers (lowest-numbered cores, so a hybrid CPU's P/E split is not what gets measured); a `PinReport` that reports what *happened* rather than what was asked, so a failed pin can never read as a pinned run; a noise floor that **fails the run** (exit non-zero) when any cell's qps spread exceeds 5%, with a compile-time assert keeping the floor under the BEN-020 margin; real kernel/governor probes, locale-independent. First use immediately refused a run — 7 cells over the floor, worst 25.6% — i.e. this machine is not quiet enough to judge G5 even pinned. **Still open: re-run the matrix on a quiet host and re-evaluate.** The bench host's power scheme is Balanced, which ramps frequency; set High performance first

The verdict doc had named it the prime suspect; optimizing it would have been the wrong layer

- [x] 1.3 **Hypothesis 2 (the client's serialized write mutex, CLT-011) CONFIRMED and fixed** (`1bdbcb6`): N concurrent calls cost N syscalls and N waiters on one lock, while the server had had drain-then-flush since T1.5 (SRV-006, Synap's measured +23%). Replaced with the server's own shape — writer task + bounded mpsc + `BufWriter` + drain-then-flush. CLT-011 is now guaranteed by there being a single writer at all, which is stronger than the mutex it replaced. Controlled A/B: pipelined-1k 1 conn **85,842 → 561,127 qps** (6.5×), p50 11,645 → 1,716 µs
